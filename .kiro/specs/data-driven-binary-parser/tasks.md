# Implementation Plan: Data-Driven Binary Parser

## Overview

This implementation plan breaks down the data-driven binary parser into incremental coding tasks. Each task builds on previous work, with property-based tests integrated close to implementation to catch errors early. The implementation uses Rust with PyO3 for Python bindings.

## Tasks

- [ ] 1. Set up module structure and core types
  - [ ] 1.1 Create `src/parser/` module directory with `mod.rs`
    - Add module declarations for definition, accessor, writer, registry, expression, error submodules
    - _Requirements: 1.1_
  - [ ] 1.2 Implement error types in `src/parser/error.rs`
    - Define LoadError, AccessError, WriteError, ConversionError, ExpressionError using thiserror
    - Include contextual fields (path, expected, actual) for all error variants
    - _Requirements: 15.1, 15.2, 15.3, 15.4, 15.5_
  - [ ] 1.3 Implement core type definitions in `src/parser/types.rs`
    - Define StructureDefinition, FieldDefinition, FieldType, Encoding, SizeSpec, RepeatSpec enums
    - Define Endian enum (Big, Little)
    - _Requirements: 1.1, 2.1, 2.2, 2.7_

- [ ] 2. Implement expression parser and evaluator
  - [ ] 2.1 Implement Expression AST in `src/parser/expression.rs`
    - Define Expression enum (Literal, FieldRef, BinaryOp, UnaryOp, MethodCall, SpecialVar)
    - Define BinaryOperator, UnaryOperator, SpecialVariable enums
    - _Requirements: 13.1, 13.2, 13.3, 13.4_
  - [ ] 2.2 Implement expression parser
    - Parse field references (dot-notation paths)
    - Parse arithmetic expressions (+, -, *, /, %)
    - Parse comparison expressions (==, !=, <, >, <=, >=)
    - Parse logical expressions (and, or, not)
    - Parse method calls (.to_i, .to_s, .length)
    - Parse special variables (_index, _root, _parent, _io)
    - _Requirements: 13.1, 13.2, 13.3, 13.4, 13.5, 13.6_
  - [ ] 2.3 Write property tests for expression parsing
    - **Property 28: Expression Arithmetic Correctness**
    - **Property 29: Expression Comparison Correctness**
    - **Property 30: Expression Logical Correctness**
    - **Property 32: Expression Syntax Error Handling**
    - **Validates: Requirements 13.2, 13.3, 13.4, 13.8**
  - [ ] 2.4 Implement ExpressionEvaluator
    - Evaluate expressions given a context with field values
    - Support type coercion (.to_i, .to_s, .length)
    - Return ExpressionError for invalid expressions
    - _Requirements: 13.5, 13.7, 13.8_
  - [ ] 2.5 Write property test for type coercion
    - **Property 31: Expression Type Coercion**
    - **Validates: Requirements 13.5**

- [ ] 3. Checkpoint - Expression evaluator complete
  - Ensure all tests pass, ask the user if questions arise.

- [ ] 4. Implement KSY definition loader
  - [ ] 4.1 Implement YAML parsing in `src/parser/definition.rs`
    - Parse meta section (id, title, endian)
    - Parse seq section into Vec<FieldDefinition>
    - Parse types section into HashMap<String, StructureDefinition>
    - Parse enums section into HashMap<String, EnumDefinition>
    - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_
  - [ ] 4.2 Implement field type parsing
    - Parse string types with size and encoding (BCS-A, BCS-N, ECS-A)
    - Parse raw byte arrays
    - Parse unsigned integer types (u1, u2, u4)
    - Parse type references to nested types
    - Parse padding attribute
    - _Requirements: 2.1, 2.2, 2.6, 2.7_
  - [ ] 4.3 Implement conditional and repetition parsing
    - Parse `if` expressions for conditional fields
    - Parse `repeat` and `repeat-expr` for expression-based repetition
    - Parse `repeat-until` for until-condition repetition
    - Parse `repeat: eos` for end-of-stream repetition
    - _Requirements: 3.1, 4.1, 4.2, 4.3_
  - [ ] 4.4 Implement definition validation
    - Validate type references resolve to defined types
    - Return UndefinedType error for missing references
    - Return LoadError for invalid YAML syntax
    - _Requirements: 1.6, 1.7_
  - [ ] 4.5 Write property tests for definition loading
    - **Property 1: Definition Round-Trip**
    - **Property 3: Invalid YAML Error Handling**
    - **Property 4: Undefined Type Reference Error**
    - **Validates: Requirements 1.1, 1.6, 1.7**

- [ ] 5. Implement character set validation
  - [ ] 5.1 Implement BCS-A, BCS-N, ECS-A validators in `src/parser/encoding.rs`
    - BCS-A: validate bytes in range 0x20-0x7E
    - BCS-N: validate bytes are digits (0x30-0x39) or space (0x20)
    - ECS-A: validate extended character set
    - _Requirements: 2.3, 2.4, 2.5_
  - [ ] 5.2 Write property tests for character validation
    - **Property 5: BCS-A Character Validation**
    - **Property 6: BCS-N Character Validation**
    - **Validates: Requirements 2.3, 2.4**

- [ ] 6. Checkpoint - Definition loader complete
  - Ensure all tests pass, ask the user if questions arise.

- [ ] 7. Implement Value type and conversions
  - [ ] 7.1 Implement Value enum in `src/parser/value.rs`

    - Use Cow<str> for zero-copy string references
    - _Requirements: 6.1, 6.5_
  - [ ] 7.2 Implement type conversion methods

    - as_i64(): parse BCS-N string as signed integer
    - as_u64(): parse BCS-N string as unsigned integer
    - as_f64(): parse numeric string as float

    - Return ConversionError for invalid conversions
    - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5, 6.6_
  - [ ] 7.3 Write property tests for value conversions
    - **Property 15: String Padding Trimming**

- [ ] 8. Implement StructureAccessor (reading)

  - [ ] 8.2 Implement offset calculation and caching

    - Handle variable-offset fields by parsing predecessors
    - _Requirements: 5.7_
  - [ ] 8.3 Implement conditional field handling
    - Evaluate condition expression to determine field presence
    - Skip conditional fields when condition is false

    - _Requirements: 3.2, 3.3, 3.4, 3.5_
  - [ ] 8.4 Write property test for conditional fields
    - **Property 7: Conditional Field Presence**
    - **Validates: Requirements 3.2, 3.3, 3.4, 3.5**
  - [ ] 8.5 Implement repetition handling
    - Parse repeated fields based on repeat-expr count
    - Parse until-condition repetitions

    - Use underscore-indexed naming (field_0, field_1, etc.)
    - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6_
  - [ ] 8.6 Write property tests for repetitions
    - **Property 8: Expression-Based Repetition Count**

    - **Property 11: Underscore-Indexed Naming**
    - **Validates: Requirements 4.1, 4.2, 4.3, 4.4, 4.5**
  - [ ] 8.7 Implement zero-copy raw data access
    - Implement raw_slice(path) returning &[u8] from original buffer
    - Implement field_byte_range(path) returning (offset, length)
    - Return NonContiguous error for non-contiguous fields
    - _Requirements: 7.1, 7.2, 7.3, 7.4, 7.5_
  - [ ] 8.8 Write property test for raw slice access
    - **Property 18: Raw Slice Identity**
    - **Validates: Requirements 7.1, 7.2**
  - [ ] 8.9 Write property tests for accessor correctness
    - **Property 12: Unknown Field Path Error**
    - **Property 13: Field Existence Correctness**
    - **Property 14: Field Enumeration Completeness**
    - **Validates: Requirements 5.3, 5.4, 5.5**

- [ ] 9. Checkpoint - StructureAccessor complete
  - Ensure all tests pass, ask the user if questions arise.

- [ ] 10. Implement StructureWriter

  - [ ] 10.5 Implement write validation

## Notes

- All tasks including property tests are required for comprehensive coverage
- Each task references specific requirements for traceability
- Property tests use the `proptest` crate with minimum 100 iterations
- Rust unit tests are inline with source using `#[cfg(test)]`
- Python tests are in `tests/` directory using pytest
- Test data goes in `data/unit/` (checked into git)
